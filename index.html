关于Node、git和Markdown文档的使用总结

*感觉Node、git以及Markdown文档是用来创建以及书写博客的，反正我所接触的就是这样，当然它们的用途可能并不仅仅指这些，但是我所接触的就是在git中输入指令来进行运行，在git中输入hexo g进行编译项目，然后再输入hexo s进行运行，并可以在本地中进行浏览，最后可以使用hexo d进行上传，以供其他人浏览 。 *

可能这三者存在着紧密的联系，彼此相互配合，相辅相成，以完成各项工作的正常进行

在进行使用之前首先要了解这些软件，了解他们的用处功能，以便可以更好地对其进行掌握。。。。

关于Node

Node.js是一正常运行个能够在服务器端运行 JavaScript 的开放源代码、跨平台 JavaScript 运行环境。Node.js 采用 Google开发的 V8 运行代码，使用事件驱动、非阻塞和 异步输入输出模型等技术来提高性能，可优化应用程序的传输量和规模。Node.js 是一个基于Chrome JavaScript 运行时建立的一个平台。这些技术通常用于数据密集的即时应用程序。 这个也是去网上进行搜索的。我本身对Node.js并没有太过于深刻的理解，可能安装这个软件就是为了一种编环境，以创造更好地编程环境来保证其他软件的正常运行。另外，我们还可以运用以下指令

$ node -v*8

v4.4.3

以确定当前Node的版本。

关于git

我有使用git来创建个人博客，我认为这是个要输入命令来进行运行的一个软件，感觉这可能是整个博客运营的一个至关重要的所在，有关博客文章的编译运行以及发布都需要运用git来完成，运用一些相关命令如hexo d等的命令。

git是一个开源的分布式版本控制系统，用以有效、高速的处理从很小到非常大的项目版本管理。这就是给它进行一个简略的定义，运用它可能就是比较高速快捷，还有一些相关的指令，如git add 命令可将该文件添加到缓存，还有git status 命令用于查看项目的当前状态，等等一系列指令。

关于Markdown文档

我接触Markdown文档是因为用来书写博客，每个软件的运行都有它们特殊的运行方法，而Markdown也有相应的运行方法Markdown是一种轻量级的标记语言，它的语法十分简单，常用的标记符号也不超过十个，设计初衷是用简洁的语法来代替排版，而不像一般我们用的字处理软件Word或Pages有大量的排版、字体设置，它使我们专心于码字，用标记语法，代替常见的排版格式，目前有很多网站支持Markdown的文字录入（例如简书），你可以导出HTML格式的文件用来网站发布，也可以十分方便的导出PDF格式的文件。当前许多网站都广泛使用 Markdown 来撰写帮助文档或是用于论坛上发表消息

每个软件都有自己的使用方法，我们要想熟练使用一种软件，就必须熟练掌握这种软件的使用方法。。。嗯，就是这样。
